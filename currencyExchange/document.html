Exchange Rate Apps Documentation

Learning Process

This is not the first time I have connected to a simple API such as this one. One this that I thought was interesting was how i used a string literal for the URL that i provided to the Fetch statement. This allowed me to very easily provide default values for the amount, and easily have a dynamic URL based on the user input. The somewhat dangerous part is that i am not currently validating the users input, which is not good practice. I don't know what type of server side validation the exchange rate API has. I am sure it is up to par, but it would be best if i had my own Client side validation and sanitation in place for both the currency codes and the dollar amount.

I struggled with getting the nodes from the DOM because i had my script tags in the header and i neglected to defer them. So it was applying the query selectors to null values. It took me WAY too long to figure that out. I need to figure out a way to remember simple things like that. I think that mostly just comes with practice i guess.

Took me a minute to remember that the best way to get data our of a fetch is to call a function inside the .then. It is by far easier than anything else i tried. It may actually be the only way to properly do it? I am not sure.
